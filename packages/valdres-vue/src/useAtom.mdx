Returns a writable `Ref` that stays in sync with the atom: reading is fresh
(read-through to the store), and writing updates the atom.

## Usage

```vue
<script setup>
import { atom } from "valdres"
import { useAtom } from "valdres-vue"

const countAtom = atom(0)
const count = useAtom(countAtom)
</script>

<template>
    <div>
        <span>{{ count }}</span>
        <button @click="count++">+</button>
    </div>
</template>
```

## Two-way binding

Since `useAtom` returns a `Ref`, it works with `v-model`:

```vue
<script setup>
import { atom } from "valdres"
import { useAtom } from "valdres-vue"

const nameAtom = atom("")
const name = useAtom(nameAtom)
</script>

<template>
    <input v-model="name" placeholder="Enter name" />
</template>
```

## Reactive (prop-driven) keys

Pass a **getter** so a family member follows a changing prop — the ref
re-subscribes (and writes route) to the new member:

```vue
<script setup>
import { useAtom } from "valdres-vue"
import { todoFamily } from "./state"

const props = defineProps(["todoId"])
const todo = useAtom(() => todoFamily(props.todoId))
</script>
```
