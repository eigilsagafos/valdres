# provideValdresScope

Scopes the current component's subtree to a child store and exposes it to
descendants — the composable form of [ValdresScope](/vue/ValdresScope), so a
`<script setup>` component can scope itself. Returns the scoped store.

```ts
interface ProvideValdresScopeOptions {
    scopeId?: string
    initialize?: InitializeCallback
}
```

## Usage

```vue
<script setup lang="ts">
import { provideValdresScope } from "valdres-vue"
import { countAtom } from "./state"

provideValdresScope({
    scopeId: "board",
    initialize: txn => {
        txn.set(countAtom, 100)
    },
})
</script>
```

Descendants now read and write the scoped store. It reuses an existing scope of
the same `scopeId` in the ancestor chain; otherwise it creates one and detaches
it on unmount. The default id is `useId()` (SSR-stable on Vue 3.5+).

## Reading scoped state in the same component

Vue's `inject` never sees a component's own provides, so this component's other
composables still resolve the **parent** store. Pass the returned store
explicitly to read scoped state here:

```vue
<script setup lang="ts">
import { provideValdresScope, useValue } from "valdres-vue"
import { filterAtom } from "./state"

const scoped = provideValdresScope({ scopeId: "list" })
const filter = useValue(filterAtom, scoped)
</script>
```

## Dynamic scope id

`scopeId` is captured once — provided context is read once by each descendant.
To re-scope on a changing id, force a remount with `:key` on the providing
component (in dev, changing it without a remount logs a warning).
