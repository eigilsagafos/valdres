A read-only composable returning a reactive `ShallowRef` for an atom or
selector. It reads **through** to the store, so the ref is fresh immediately
after a `store.set` — consistent with [useAtom](/vue/useAtom), even under the
default batched store. (A chained `computed()` stays microtask-deferred — this
is sibling consistency, not fully synchronous propagation.)

## Usage

```vue
<script setup>
import { atom, selector } from "valdres"
import { useValue } from "valdres-vue"

const nameAtom = atom("Valdres")
const greetingSelector = selector(get => `Hello, ${get(nameAtom)}!`)

const greeting = useValue(greetingSelector)
</script>

<template>
    <h1>{{ greeting }}</h1>
</template>
```

## Reactive (prop-driven) keys

Pass a **getter** so a family member follows a changing prop — the composable
re-subscribes when the resolved member changes:

```vue
<script setup>
import { useValue } from "valdres-vue"
import { todoFamily } from "./state"

const props = defineProps(["todoId"])
const todo = useValue(() => todoFamily(props.todoId))
</script>
```

## Async selectors

`useValue` **never throws** on a pending async selector (or a promise atom
default): the ref reads `undefined` while pending and holds the last resolved
value once available. On a key change it holds the previous value until the new
state resolves. For pending/error status or `<Suspense>` support, use
[useAsyncValue](/vue/useAsyncValue).
