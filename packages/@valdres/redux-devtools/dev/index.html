@valdres/redux-devtools

Drive a valdres store and watch every change flow into the Redux DevTools extension — including scopes and time-travel.

Checking for the Redux DevTools extension…

Open the Redux DevTools panel (browser devtools → Redux tab), pick the valdres-playground instance, then click around. Use the panel's slider / "Jump" to time-travel — the page reflects the restored state live. The store is enumerable, so the timeline opens already seeded with the initial state (count: 3, a message, a todo) rather than empty.

doubleCount, countParity, todoCount, messageLength, and summary are selectors — pure values derived from the atoms above. They recompute automatically (no setter) and appear under @computed in DevTools. summary derives from other selectors, so changing count cascades through both. user/isDark is subscribed inside sessionA, so it lands under that scope: @scopes.sessionA.@computed.user/isDark (root selectors stay in the top-level @computed).
