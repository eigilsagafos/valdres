# getValdresContext

Returns the store set by the nearest [setValdresContext](/svelte/setValdresContext) / [scope](/svelte/scope) ancestor. Most of the time you don't need it — [fromState](/svelte/fromState), [toStore](/svelte/toStore), and [transaction](/svelte/transaction) resolve the context store on their own. Reach for it when you need direct `store.get` / `store.set` access.

## Usage

```svelte
<script lang="ts">
    import { getValdresContext } from "valdres-svelte"
    import { countAtom } from "$lib/state"

    const store = getValdresContext()
    console.log(store.get(countAtom))
</script>
```

<div className="callout callout-note">
  <div className="callout-title">Call during initialization</div>
  Svelte throws <code>lifecycle_outside_component</code> for <code>getContext</code> outside component initialization, so capture the store at the top level rather than inside an event handler. To run writes from a handler, use <a href="/svelte/transaction">transaction</a>, which captures the store for you.
</div>
